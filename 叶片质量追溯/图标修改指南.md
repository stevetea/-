# 图标修改指南

## 📋 概述

本指南说明如何修改叶片质量追溯系统的应用图标。

## 🎯 图标文件位置

图标文件位于：`static/logo.png`

## 📝 修改步骤

### 1. 准备图标文件

- **文件格式**：PNG（推荐）或 JPG
- **推荐尺寸**：
  - 主图标：512x512 或 1024x1024 像素
  - 确保图标清晰，背景透明（PNG格式）
- **文件大小**：建议小于 500KB

### 2. 替换图标文件

将新图标文件重命名为 `logo.png`，替换以下文件：
```
yepian/叶片质量追溯/static/logo.png
```

### 3. 不同平台的图标配置

#### ✅ H5 版本（已完成配置）

已在 `index.html` 中配置 favicon：
```html
<link rel="icon" type="image/png" href="/static/logo.png" />
```

#### ✅ App 版本（已完成配置）

已在 `manifest.json` 中配置图标路径：
- Android：支持多种尺寸（36x36 到 192x192）
- iOS：支持 iPhone 和 iPad 多种尺寸

#### ⚠️ 微信小程序（需要手动配置）

微信小程序的图标需要在**微信开发者工具**中配置：

1. 打开微信开发者工具
2. 点击右上角"详情"
3. 在"本地设置"或"项目设置"中找到"AppID"和"项目配置"
4. 在"上传"或"预览"时，系统会提示配置图标
5. 或者登录[微信公众平台](https://mp.weixin.qq.com)：
   - 进入小程序管理后台
   - 设置 → 基本设置 → 小程序头像
   - 上传图标（建议尺寸：512x512，PNG格式）

**注意**：微信小程序图标要求：
- 尺寸：512x512 像素
- 格式：PNG
- 文件大小：不超过 2MB
- 不能使用透明背景

## 🔄 重新编译

修改图标后，需要重新编译项目：

### H5 版本
```bash
npm run dev:h5
# 或
npm run build:h5
```

### 微信小程序
```bash
npm run dev:mp-weixin
# 或
npm run build:mp-weixin
```

### App 版本
在 HBuilderX 中：
1. 右键项目 → 发行 → 原生App-云打包
2. 或 发行 → 原生App-本地打包

## 📱 验证图标

### H5 版本
- 打开浏览器，查看浏览器标签页的图标
- 清除浏览器缓存后刷新页面

### 微信小程序
- 在微信开发者工具中预览
- 或上传体验版后，在微信中查看

### App 版本
- 打包后安装到手机查看
- 查看桌面图标和应用内图标

## 🎨 图标设计建议

1. **简洁明了**：图标应该清晰易识别
2. **尺寸适配**：确保在不同尺寸下都清晰
3. **品牌一致性**：与系统整体风格保持一致
4. **避免文字**：小尺寸下文字可能看不清
5. **颜色对比**：确保在浅色和深色背景下都清晰可见

## 📂 相关文件

- 图标文件：`static/logo.png`
- H5配置：`index.html`
- App配置：`manifest.json`
- 小程序配置：需要在微信开发者工具或微信公众平台配置

## ⚠️ 注意事项

1. **文件路径**：确保图标文件路径正确
2. **文件格式**：推荐使用 PNG 格式
3. **缓存问题**：修改后可能需要清除浏览器/小程序缓存
4. **微信小程序**：图标修改后需要重新上传代码
5. **App版本**：图标修改后需要重新打包

## 🔧 故障排查

### 图标不显示？

1. **检查文件路径**：确认 `static/logo.png` 文件存在
2. **检查文件格式**：确保是有效的图片文件
3. **清除缓存**：清除浏览器或小程序缓存
4. **重新编译**：重新编译项目
5. **检查配置**：确认相关配置文件已正确更新

### 图标模糊？

1. **使用高分辨率图标**：建议使用 1024x1024 或更高分辨率
2. **检查压缩**：确保图标没有被过度压缩
3. **使用矢量图**：如果可能，使用 SVG 格式（需要转换为 PNG）

## 📞 需要帮助？

如果遇到问题，请检查：
- 文件路径是否正确
- 文件格式是否支持
- 是否已重新编译项目
- 是否已清除缓存
